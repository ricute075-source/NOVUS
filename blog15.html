I. Giới thiệu về ngăn xếp MERN

1. MERN là gì? (MongoDB, Express, React, Node.js)

**MERN** là một ngăn xếp công nghệ (tech stack) dựa trên JavaScript được sử dụng để xây dựng các ứng dụng web full-stack. Tên gọi MERN là từ viết tắt của bốn công nghệ cốt lõi:

  • MongoDB: Một cơ sở dữ liệu NoSQL, hướng tài liệu (document-oriented), giúp lưu trữ dữ liệu dưới dạng các đối tượng JSON linh hoạt.
  • Express.js: Một framework backend tối giản và linh hoạt cho Node.js, được sử dụng để xây dựng các API web.
  • React.js: Một thư viện JavaScript phía client (frontend) để xây dựng giao diện người dùng (UI) tương tác dựa trên các component.
  • Node.js: Một môi trường thực thi JavaScript phía máy chủ (backend), cho phép chạy JavaScript bên ngoài trình duyệt.
Sự kết hợp này cho phép các nhà phát triển xây dựng ứng dụng hoàn chỉnh chỉ bằng một ngôn ngữ lập trình duy nhất là **JavaScript**.

2. Tại sao chọn MERN cho dự án của bạn?

Ngăn xếp MERN ngày càng trở nên phổ biến nhờ vào nhiều ưu điểm vượt trội. Thứ nhất, việc sử dụng **JavaScript cho cả frontend và backend** giúp đơn giản hóa quá trình phát triển và cho phép các nhà phát triển dễ dàng chuyển đổi giữa hai vai trò. Thứ hai, **React** cung cấp một cách tiếp cận mạnh mẽ để xây dựng giao diện người dùng động và hiệu suất cao.

Thứ ba, **MongoDB** với cấu trúc dữ liệu linh hoạt rất phù hợp với các ứng dụng hiện đại, nơi các yêu cầu về dữ liệu có thể thay đổi nhanh chóng. Cuối cùng, cộng đồng hỗ trợ lớn mạnh và hệ sinh thái phong phú của Node.js và React đảm bảo rằng bạn luôn có thể tìm thấy các thư viện và công cụ cần thiết để giải quyết bất kỳ vấn đề nào.

II. Xây dựng Backend với Node.js và Express

1. Thiết lập dự án và cài đặt dependencies

Đầu tiên, hãy tạo một thư mục cho dự án backend và khởi tạo một dự án Node.js mới bằng lệnh `npm init -y`. Sau đó, cài đặt các thư viện cần thiết: 

npm install express mongoose cors dotenv
  • `express`: Framework web.
  • `mongoose`: Thư viện để tương tác với MongoDB.
  • `cors`: Middleware để cho phép các yêu cầu từ tên miền khác (từ frontend React).
  • `dotenv`: Để quản lý các biến môi trường.

2. Tạo API endpoints với Express

Tạo một tệp `server.js` và thiết lập một máy chủ Express đơn giản. Bạn sẽ định nghĩa các **routes** (tuyến đường) để xử lý các yêu cầu HTTP (GET, POST, PUT, DELETE) từ client. Ví dụ, bạn có thể tạo một endpoint `/api/products` để trả về danh sách sản phẩm. 


// server.js
const express = require('express');
const app = express();

app.get('/api/products', (req, res) => {
  // Logic để lấy sản phẩm từ database
  res.json([{ id: 1, name: 'Sản phẩm A' }]);
});

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => console.log(`Server running on port ${PORT}`));

3. Kết nối với cơ sở dữ liệu MongoDB

Sử dụng Mongoose để kết nối với cơ sở dữ liệu MongoDB của bạn. Bạn sẽ định nghĩa các **Schema** (lược đồ) để mô tả cấu trúc dữ liệu và các **Model** để tương tác với các collection trong database. 


// models/Product.js
const mongoose = require('mongoose');

const ProductSchema = new mongoose.Schema({
  name: { type: String, required: true },
  price: { type: Number, required: true },
});

module.exports = mongoose.model('Product', ProductSchema);
Sau đó, bạn có thể sử dụng model `Product` trong các route của mình để thực hiện các thao tác CRUD (Create, Read, Update, Delete) với cơ sở dữ liệu.

III. Xây dựng Frontend với React

1. Khởi tạo dự án React với Create React App

**Create React App** là cách nhanh nhất để bắt đầu một dự án React mới với cấu hình được tối ưu hóa sẵn. Mở một terminal mới và chạy lệnh: 

npx create-react-app client
Lệnh này sẽ tạo một thư mục `client` chứa toàn bộ cấu trúc dự án React.

2. Tạo các Components và quản lý State

Trong React, giao diện người dùng được xây dựng từ các **components** có thể tái sử dụng. Bạn có thể tạo các component cho từng phần của ứng dụng, ví dụ: `ProductList`, `ProductItem`, `Navbar`.

Để quản lý dữ liệu và trạng thái của ứng dụng, React sử dụng khái niệm **state**. Bạn có thể sử dụng hook `useState` để lưu trữ trạng thái cục bộ trong một component và `useEffect` để thực hiện các side effect, chẳng hạn như gọi API.

3. Gọi API từ Backend

Để lấy dữ liệu từ backend, bạn có thể sử dụng `fetch` API của trình duyệt hoặc một thư viện phổ biến như **Axios**. Ví dụ, trong component `ProductList`, bạn có thể gọi đến endpoint `/api/products` đã tạo ở backend để hiển thị danh sách sản phẩm. 


// components/ProductList.js
import React, { useState, useEffect } from 'react';
import axios from 'axios';

function ProductList() {
  const [products, setProducts] = useState([]);

  useEffect(() => {
    const fetchProducts = async () => {
      const { data } = await axios.get('/api/products');
      setProducts(data);
    };
    fetchProducts();
  }, []);

  // ... render danh sách sản phẩm
}

4. Routing trong ứng dụng React

Để tạo một ứng dụng đa trang (Single Page Application - SPA), bạn sẽ cần một thư viện routing. **React Router** là lựa chọn phổ biến nhất. Nó cho phép bạn định nghĩa các route để hiển thị các component khác nhau tương ứng với các URL khác nhau trên trình duyệt.

IV. Triển khai ứng dụng

Sau khi hoàn thành việc phát triển, bước cuối cùng là triển khai ứng dụng để người dùng có thể truy cập. Có nhiều nền tảng để triển khai ứng dụng MERN, ví dụ:

  • Heroku: Một nền tảng PaaS (Platform as a Service) phổ biến, dễ sử dụng cho cả backend và frontend.
  • Vercel/Netlify: Rất mạnh mẽ và tối ưu cho việc triển khai các ứng dụng frontend hiện đại như React.
  • DigitalOcean/AWS/Google Cloud: Cung cấp các máy chủ ảo (VPS) cho phép bạn toàn quyền kiểm soát môi trường triển khai.
Quá trình triển khai thường bao gồm việc build phiên bản production của ứng dụng React, cấu hình các biến môi trường và khởi chạy máy chủ Node.js.
